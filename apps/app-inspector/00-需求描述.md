# 需求描述

## 目标

用对抗式多角色协作，产出一个**可视化看板程序**。

这个看板程序的用途：输入任意 app.yaml 文件，生成一张交互式拓扑图，帮助开发者审核 app.yaml 的架构质量——一眼看出角色流转关系、物料输入输出关系、知识注入关系。

## 看板核心价值

用户打开看板就能看到：
1. **角色流转图**：角色按 DAG 拓扑分层排列，标注 verdict 路由边
2. **物料文件树**：knowledge / workspace / external 三类文件按目录树展示
3. **悬停交互**：选中角色时，绿色高亮 input 注入、紫色高亮 knowledge 注入、红色高亮 output 产出，其余变暗
4. **死链/孤儿标注**：有问题的文件和角色有视觉警告
5. **缩放平移**：支持滚轮缩放和拖拽平移

## 对抗设计

- **功能明确者** 分析看板需要哪些内容 → **功能红队** 审查够不够清晰 → **人工审阅** 用户拍板 → **功能实现者** 编码实现 → **网页红队** 截图验收

## 最终产物

- `outputs/功能规格书.md`：看板的功能规格书
- `outputs/dashboard.html`：自包含的可视化看板程序
