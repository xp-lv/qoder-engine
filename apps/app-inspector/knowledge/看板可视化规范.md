# 看板可视化规范

## 1. 目标

生成的看板程序（自包含 HTML）必须实现以下功能：

### 1.1 DAG 拓扑图
- 角色按**拓扑分层**自上而下排列（Kahn's algorithm）
- 同层角色水平排列
- 角色→角色的 verdict 边用弧线连接，标注 verdict 值
- 不同边类型用不同样式：正常边(实线)、回退边(红色粗线)、循环边(黄色虚线)

### 1.2 物料文件树
- knowledge 文件在左侧按目录树展开
- workspace 文件在右侧按目录树展开
- external 文件在顶部横排
- 每个文件标注 ✅(存在) / ❌(缺失)

### 1.3 悬停交互
- **悬停角色节点**：
  - 该角色的 input 文件 + 连线高亮为**绿色脉冲动画**
  - 该角色的 knowledge 注入文件 + 连线高亮为**紫色脉冲动画**
  - 该角色的 output 文件 + 连线高亮为**红色脉冲动画**
  - 其余所有节点和连线变暗(opacity 0.15)
  - Tooltip 显示：skill 路径、输入/输出文件、verdict、路由出/入边
- **悬停文件节点**：
  - Tooltip 显示：文件路径、类型、谁生产它、谁消费它

### 1.4 缩放与平移
- 滚轮缩放（0.15x ~ 3x）
- 鼠标拖拽平移
- Reset 按钮

## 2. 死链/孤儿标注

在生成的看板中，对死链和孤儿物料必须有**视觉标注**：
- 死链文件：红色边框 + ⚠️ 图标
- 孤儿物料：橙色边框 + 问号图标
- 断路角色：红色虚线边框

## 3. 技术约束

- **自包含**：单个 HTML 文件，不依赖外部资源（CDN 可选）
- **暗色主题**：背景 `#0f172a`，文字 `#e2e8f0`
- **SVG 渲染**：使用原生 SVG，不用 Canvas
- **中文字体**：`-apple-system, 'PingFang SC', sans-serif`

## 4. 产出物 schema

功能实现者产出的 HTML 文件必须包含以下数据结构（嵌入 `<script>` 中）：

```javascript
const ROLES = {
  "r0_0": {
    name: "角色名",
    confirm: "manual|auto",
    skill: "roles/角色名/skill.md",
    inputs: [{name, path, exists}],
    outputs: [{name, path, exists}],
    knowledge: [{name, path, exists}],
    verdicts: ["v1", "v2"],
    out_edges: [{v, t}],
    in_edges: [{f, v}]
  }
};
const FILES = {
  "kn_0": {
    name: "文件名",
    path: "knowledge/xxx.md",
    cat: "knowledge|workspace|external",
    producers: ["角色A"],
    consumers: ["角色B"]
  }
};
```

## 5. 质量验收标准

网页红队检查看板程序时，以下条件全部满足才判 `passed`：
1. 角色按 DAG 拓扑分层（不是平铺列表）
2. knowledge/workspace/external 按文件树展示（不是平铺）
3. 悬停角色有绿色注入 + 红色产出动画
4. 死链/孤儿物料有视觉标注
5. 支持缩放和平移
6. 所有角色和文件都有 tooltip

## 6. 正反例对照

### 正例（合格看板的可见特征）
- 角色节点按 L0→L1→L2 自上而下分层，同层水平排列
- 左侧紫色区域显示 knowledge/ 目录树，右侧绿色区域显示 outputs/ 目录树
- 悬停角色时，input 连线呈绿色脉冲闪烁，output 连线呈红色脉冲闪烁，其余节点变暗
- 死链文件节点有红色边框 + 警告图标
- 滚轮可缩放（0.15x~3x），拖拽可平移

### 反例（常见错误）
- 角色平铺为一行/一列，不按拓扑分层
- 所有文件混在一起，不分 knowledge/workspace/external
- 悬停角色无任何视觉变化（未实现高亮动画）
- 无缩放/平移功能，大 app 看不全
- 文件存在性未标注（无法区分已有/缺失）
