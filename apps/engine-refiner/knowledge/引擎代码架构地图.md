## 4. 关键数据流

### 4.1 编译流
```
app.yaml → compiler.py → ROUTER.json + registry.json + manifest.json + roles/ 骨架
```

### 4.2 执行流（指令周期）
```
用户/Hook② → step.py --next
  → orchestrator.py --phase dispatch
    → router.py（沿 DAG 边找下一个 step）
    → 检查 input_groups 收敛
  → 返回 dispatch 指令

role-executor 执行 → step.py --submit
  → orchestrator.py --phase post_execute
    → gate.py（校验产出物）
    → router.py（推进状态）
  → 返回 next: dispatch/confirm/complete
```

### 4.3 状态写入流
```
任何 STATE 修改 → state_io.state_txn()
  → acquire_lock() (filelock.py)
  → load_state()
  → [修改]
  → _write_unlocked() (tempfile + os.replace 原子写入)
  → release_lock()
```

## 5. 已知代码质量问题区域（优先关注）

| 区域 | 问题类型 | 说明 |
|---|---|---|
| state_io.py:42-51 | 效率 | 生产写入路径嵌入诊断日志（traceback 提取 + 文件追加） |
| compiler.py | 简洁 | 1482 行单文件，parse_edges_line 等函数过长 |
| router.py:main() | 优雅 | 300+ 行单体函数 |
| orchestrator.py | 一致性 | 版本注释散落（v9.1/v9.2/v4.2 混用） |
| output() 函数 | 一致性 | 同名函数在 router.py、orchestrator.py、step.py 中重复定义 |
| filelock.py | 一致性 | 跨平台重复函数名（acquire_lock/release_lock 出现两次） |
