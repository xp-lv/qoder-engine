# 对抗校验准则

> 注入目标：忠实度红队（M6）、完整性红队（M7）、一致性红队（M8）
> 用途：定义三维语义对抗校验的维度边界、problem 分级标准与判定规则

## 目的
本准则定义 app-builder-v2 的 L4 校验层语义对抗子阶段中，三红队各自聚焦的单一维度、problem severity 分级标准与跨维度隔离原则，消除 v1 中"每个红队三维重复检查"的冗余。

## 核心准则

### 1. 三维独占原则（蓝图 §3.5，P-R4-002 修订）
三红队各聚焦单一维度，不在多模块重复实现同一检查逻辑：
| 红队 | 聚焦维度 | 校验问题 | 不负责 |
|------|----------|----------|--------|
| M6 忠实度红队 | **忠实度** | skill 内容是否忠实于蓝图角色职责描述 | 完整性、一致性 |
| M7 完整性红队 | **完整性** | 蓝图声明的角色/knowledge/产出物是否都已生成 | 忠实度、一致性 |
| M8 一致性红队 | **一致性** | skill 间引用路径/verdict 名称是否一致 | 忠实度、完整性 |

### 2. 与 M5 编译预检师的层级隔离（蓝图 §3.5，P-R4-008 修订）
M5 与三红队三重隔离，无维度重叠：
- **校验对象**：M5 校验结构骨架（4 个结构文件），红队校验内容语义（skill/knowledge 文本）
- **校验性质**：M5 机械编译（compiler.py --check），红队对抗深度（语义分析）
- **判定输出**：M5 二元 pass/fail，红队三档 grading（high/medium/low problem）

### 3. problem severity 分级标准
| severity | 定义 | 判据 | 典型示例 |
|----------|------|------|----------|
| **high** | 阻断性缺陷，APP 包无法运行或核心能力缺失 | 角色缺技能文件 / verdict 路由断裂 / 蓝图角色完全未实现 | skill.md 缺失、ROUTER.json 有死链 |
| **medium** | 降级性缺陷，APP 包可运行但质量不达标 | skill 内容部分偏离蓝图 / knowledge 内容空洞 | skill 缺自检项、knowledge < 20 行 |
| **low** | 改进性建议，不影响运行但影响可维护性 | 命名不统一 / 注释缺失 / 格式不规范 | verdict 描述措辞不一致 |

### 4. 红队 verdict 语义
红队的 verdict 仅表示"校验工作完成"，不直接驱动回退：
- `confirmed`：校验完成，问题清单已产出（无论是否发现问题）
- 问题清单的 severity 分布由 M9 终审裁决者读取后统一裁决
- 红队不直接 emit fail 类 verdict（回退决策归 M9）

### 5. Devil's Advocate 对抗原则
红队应主动扮演"魔鬼代言人"（Devil's Advocate），在最恶劣条件下测试 APP 包：
- 不满足于"看起来没问题"，要构造反例
- 对每个声明都要问"如果这个声明是错的会怎样"
- 高 severity 问题必须有可复现的证据（具体文件 + 行号 + 反例）

## 判别清单

三红队校验时各自核对（仅本维度）：

### M6 忠实度红队
- [ ] 每个角色的 skill"角色定位"忠实于蓝图独特能力声明？
- [ ] 每个角色的 skill"执行步骤"覆盖蓝图核心职责？
- [ ] 每个角色的 skill"verdict 规则"与蓝图拓扑出边一致？

### M7 完整性红队
- [ ] 蓝图 §3.1 每个角色都有 skill.md + schema.json？
- [ ] 蓝图 §3.6 每个 knowledge 引用都有对应文档？
- [ ] 蓝图 §3.2 每类产出物都有负责角色？

### M8 一致性红队
- [ ] 跨角色 inputs/outputs 路径对齐？
- [ ] ROUTER.json verdict 与 schema.json verdict enum 一致？
- [ ] 角色名在 app.yaml/registry.json/ROUTER.json 三处统一？

## 反模式
- ❌ 红队复核非本维度问题（M6 检查完整性、M7 检查一致性等）——违反三维独占
- ❌ 红队直接 emit fail 类 verdict 驱动回退——回退决策归 M9
- ❌ problem 缺 severity 分级或缺修复建议——M9 无法据此裁决
- ❌ 红队只报喜不报忧（confirmed 但问题清单为空却实际有问题）——违反 Devil's Advocate
